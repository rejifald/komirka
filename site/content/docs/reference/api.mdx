> The package is **tessellum** (published on npm as a 0.0.0 placeholder; repo
> [github.com/rejifald/tessellum](https://github.com/rejifald/tessellum)). The primitive is
> **`tessera`** (one mosaic tile — one config value; see
> [naming](/docs/reference/decisions)). This page is the canonical contract: every example
> elsewhere on this site uses only what is defined here.

## Entry points

```ts
import { tessera, defineTesserae, refresh, identityOf, keepLastGood, isSecret } from "tessellum";
import { literal, chain, provider } from "tessellum";         // universal wiring combinators (pure data)
import type { Tessera, Secret, Snapshot, LiveBinding, AsyncBinding, Scope, ConfigOf } from "tessellum";

// pick ONE bind per entrypoint — the entry point is what grants capability:
import { bind, env, file } from "tessellum/node";     // natural defaults = process.env + fs
// import { bind } from "tessellum/workers";          // per-request bag REQUIRED at opts: bind(entries, { env })
// import { bind } from "tessellum/async";            // async-capable (providers); re-exports node combinators
// import { bind } from "tessellum";                  // core: no natural defaults — literal() entries only

// tests: a dedicated entry point of pre-certified doubles (see Testing):
import { secretLiteral, fakeSecretProvider, manualClock, verifyProvider } from "tessellum/testing";
```

The core entry (`"tessellum"`) is universal — it runs in browsers, Workers, Node, Deno, and
Bun, and it ships **no source resolvers**: core `bind` supplies no natural defaults (a
bare `env`/`file` ref under it is `NoWiringError`; only `literal()` entries resolve).
`env`/`file` live in `tessellum/node`, which imports `node:process`/`node:fs` so bundling it
for a browser fails at build. Capability is granted only by the bind entry point — never
by wiring data, which is pure JSON-representable description.

## `tessera(config)` — the descriptor

```ts
type Json = string | number | boolean | null | Json[] | { [k: string]: Json };

type SourceRef =
  | { env: string }                     // environment variable
  | { file: string }                    // file contents (re-readable — may be live)
  | { provider: string; key?: string }  // named provider supplied at bind
  | { literal: Json };                  // inline value

interface TesseraConfig {
  key: string;                          // canonical name            (identity)
  schema?: StandardSchemaV1;            // BYO validator — zod/valibot/arktype
  doc?: string;                         // human documentation       (cosmetic)
  exposure?: "secret" | "public";       // DEFAULT "secret"          (identity)
  sources?: SourceRef[];                // ordered precedence        (identity)
                                        //   default: [{ env: key }]
  default?: Json;                       // JSON only                 (identity)
  freshness?: "snapshot" | "live";      // DEFAULT "snapshot"
  version?: string;                     // schema-identity discriminator
}

function tessera(config: TesseraConfig): Tessera<T>;
// T inferred from the schema's OUTPUT: z.coerce.number() -> Tessera<number>.
// No schema -> Tessera<string>.
```

Construction is **inert**: zero I/O, zero validation, zero registration, zero caching. The
result is frozen, carries a structural `$tessera` brand (never checked via `instanceof`), and
round-trips as JSON minus the schema function. A non-JSON value in any field is a
`TypeError` at construction.

```ts
function defineTesserae<T extends readonly Tessera[]>(tesserae: T): T;
// const-tuple inference for groups; groups are plain arrays declared near call sites
```

## Binding — where values live

```ts
// One generic signature per door (TS >= 5.0 `const` type parameters — the floor).
// Entries: a single tessera/variants group, an array, or a record. Sources: one or many.
// The snapshot mirrors the input shape:
//   bind(dbUrl, src).value                 : string
//   bind([dbUrl, poolSize], src).values    : readonly [string, number]  (real frozen tuple)
//   bind({ db: dbUrl }, src).values.db     : string  (record under YOUR names;
//                                            variants entries yield the match() union)
// Snapshots are deliberately NOT iterable (positional iterable types are a lie in TS).
// .value/.values are ABSENT on LiveBinding/AsyncBinding (freshness is a type parameter;
// widened Tessera<string> fails safe to no-sugar) + runtime LivenessViolationError backstop.
// The values bag: null-prototype, redacting toJSON + inspect (secrets never dump via
// JSON.stringify/console.log); spread = unwrap() on every tessera; ReservedKeyError for
// record keys "toJSON" | "then" | "constructor".

// There is NO sources argument: wiring is per entry, and bare entries resolve their
// DECLARED refs against the bind entry point's natural defaults.
type Entry = Tessera | VariantsGroup | WiredEntry;
type BindInput = Entry | readonly Entry[] | Record<string, Entry>;

// EAGER (the default door): validates the listed set NOW — one aggregated MissingConfigError.
function bind<const K extends BindInput>(entries: K, opts?: BindOptions): BindResult<K>;
bind.lazy(entries, opts?);       // HALF-LAZY: presence eager, validation deferred
bind.unchecked(entries, opts?);  // fully deferred — the name carries the warning

// WIRING COMBINATORS (pure JSON-representable data — capability enters only at bind):
env(tessera, "PLATFORM_NAME")   // REF RENAME: patches the matching declared ref IN PLACE,
file(tessera, "/etc/app/x")     //   preserving declared fallbacks; UnknownWiringError if the
provider(tessera, impl, key?)   //   tessera declares no ref of that kind. Name/path REQUIRED.
chain(tessera, provider(ssm, "/prod/sg"), env("SG_KEY"))
                             // the ONLY full-replacement spelling — errors list dropped refs;
                             //   links carry color: one async link → AsyncBinding
env(group, { prefix: "MAILER_", overrides: { password: "SECRET_PW" } })
                             // group wiring: ONE entry per tessera ever (DuplicateWiringError);
                             //   overrides escape the prefix; the discriminant is a member
literal(tessera, inputValue)    // input-typed; validation RUNS; exposure redaction applies
                             //   (replaces snapshotOf — testing = all-literal entries)

// COLORING: the result type is computed from the entries —
//   any async entry → AsyncBinding; else any live tessera → LiveBinding; else Snapshot.
// Provider impls declare { id, color: "sync" | "async" } — the id is what the lockfile
// records; anonymous impls are a bind error on bake-covered targets. There is no load():
//   const cfg = await bind(entries).snapshot();   // resolve once → sync reads forever

interface BindOptions {
  env?: EnvBag;                  // workers: the per-request bag — REQUIRED there, never wiring data
  transform?: { prefix?: string; rename?: Record<string, string> };
                                 // app-wide, ONLY as a restatement of manifest-declared data
                                 // (digest-verified → WiringSkewError). LAW: per-entry/group
                                 // overrides are absolute; rename is terminal; prefix for the rest.
  maxStalenessMs?: number;       // live/async poll-on-read staleness budget (staleness of the SERVED value)
  readTimeoutMs?: number;        // per-read WAIT deadline (default: maxStalenessMs) — bounds every blocking
                                 //   re-read; breach → SourceUnavailableError, never a stale serve (ADR 0005)
  readyTimeoutMs?: number;       // ready() boot budget (default 30_000): bounded retry w/ backoff, each
                                 //   attempt ≤ readTimeoutMs, then fail-fast; 0 = die on first strike
  onInvalid?: (tessera: Tessera, raw: string, issues: Issue[]) => void; // once per (tessera, distinct raw); secrets get { rawDigest, length }
  onSourceError?: (tessera: Tessera, cause: unknown) => void;           // never routed through onInvalid
  invalid?: KeepLastGood[];      // keepLastGood(tessera, { maxCoastMs }) — maxCoastMs required
  unify?: { accept: Tessera; aliases: string[] }[];                  // identity alias table (see Identity)
}
```

**One runtime shape, three type views.** Every binding implements the union surface —
`ready()` is pre-resolved on snapshots, `epoch` is frozen at 0, `unwrap` is universal, and
`set()` exists but throws `NotWritableError` where unsupported. `Snapshot`/`LiveBinding`/
`AsyncBinding` only narrow what you *may* call, so the widening fail-safe is safe at
runtime, not just in the type system. Validators are still checked at every validation
site for input-dependent async-ness (`AsyncValidatorInSyncBindingError`). Every read comes
as a pair: `unwrap(k)` throws on failure, and `safe(k)` is its non-throwing twin —
returning `{ ok: false, error }` with exactly the `ConfigReadError` `unwrap(k)` would have
thrown (validation failures carry `.issues`).

## `Snapshot` — pinned, immutable

```ts
interface Snapshot {
  unwrap<T>(k: Tessera<T>): T;          // strict read, typed. Three outcomes: value |
                                     // UnboundTesseraError | DescriptorMismatchError
  safe<T>(k: Tessera<T>): { ok: true; value: T } | { ok: false; error: ConfigReadError };
                                     // non-throwing twin of unwrap(); error carries .issues on validation failure
  pick<T extends readonly Tessera[]>(tesserae: T): Scope<ConfigOf<T>>;
  explain(k: Tessera): Provenance;      // source, resolved key, version? (the provider CAS token
                                     //   this pin carries); secret VALUE redacts, the token does not
  toJSON(): Json;                    // secrets redacted by construction
}
```

- `Snapshot` is the only type cross-package APIs should accept. It never changes under you.
- `pick()` **runtime-restricts**: the returned `Scope` contains only the picked entries;
  reading anything else throws `NotInScopeError`. The `Scope` type is brand-gated — only
  `pick()` produces it, and the root `Snapshot` is deliberately not assignable to it, so
  library signatures force the narrowing.
- A **secret**-exposure tessera's `T` is `Secret<V>` (`V` = the schema output): assignable to
  `V`, so usable anywhere a `V` is, but it throws `ExposureError` on `JSON.stringify` and is
  non-cloneable (`structuredClone`/`postMessage` throw `DataCloneError`) — closing the
  request-time SSR/RSC payload class no build-time scan can see. Coercion works everywhere;
  call `.reveal(): V` at a boundary that *typechecks* its input (`crypto`,
  `z.string().parse`), and `isSecret(v)` to detect one. `tessera()` brands the type when
  `exposure` is secret (the default); public tesserae are plain `V`. Full contract:
  [ADR 0002](https://github.com/rejifald/tessellum/blob/main/docs/adr/0002-secret-value-representation.md).

## `LiveBinding` — poll-on-read freshness

```ts
interface LiveBinding {
  unwrap<T>(k: Tessera<T>): T;     // re-reads raw (throttled per-source maxStalenessMs);
                                // re-validates ONLY when the raw value changed
  safe<T>(k: Tessera<T>): { ok: true; value: T } | { ok: false; error: ConfigReadError };
  snapshot(): Snapshot;         // THE per-request pin; single coherent pass per source
  epoch(k: Tessera): number;       // monotonic; bumps when an accepted value changes
  health(): HealthReport;       // per-tessera staleness / last validation error
  pick(tesserae): Scope;           // pick() pins — a Scope is always a consistent cut
}

async function refresh(b: LiveBinding | Snapshot, opts?: { partial?: "keep-old" }): Promise<Snapshot>;
// Returns a NEW pinned snapshot; never mutates in place.
// All-or-nothing on partial source failure ({ partial: "keep-old" } is the opt-in).
// Concurrent refresh() calls on one binding are single-flight coalesced.
```

Live is only meaningful for **re-readable** sources — files and providers. A running
process's OS environment never changes, so `freshness: "live"` over an `{ env }` source
draws a dev-mode warning (and on workerd, where env is fixed per deployment, live is a
no-op).

**When a live re-read turns invalid, current truth wins** (no inherited values): the tessera
becomes *poisoned* — `snapshot()` never throws wholesale (the bad tessera occupies an error
slot), `unwrap(thatTessera)` throws `InvalidLiveValueError`, `safe()` is the non-throwing
form, and `epoch` bumps on the transition (single clock — epoch consumers use `safe`).
Invalid is not unavailable: a source that is *down* serves the cached value within
`maxStalenessMs` (the declared budget), then `SourceUnavailableError`. Keep-last-good is
a bind-site opt-in — `keepLastGood(tessera, { maxCoastMs })` — for values where availability
beats freshness; past the coast budget it throws `CoastBudgetExceededError`. A tessera that
has never had a valid value throws `NoValidValueError`.

## Conditional config — `variants()`, `when()`, `rule()`

```ts
function variants<D extends Tessera<string>, B>(on: D, branches: B): Variants<D, B>;
// bind resolves the discriminant FIRST (eagerly, in every bind mode), then requires
// and validates ONLY the active branch. Branches may reuse the same env key with
// different schema/default — sibling branches never co-bind.
snapshot.match(email)
// → { type: "console", values: {} } | { type: "smtp", values: { host: string; … } } | …
// match() exists on Snapshot and eager bindings only — pin LiveBinding/AsyncBinding first.

function when(cond: { tessera: Tessera; is: "set" | { equals: Json } | { in: Json[] } },
              members: Record<string, Tessera>): When;
// "set" = a source produced a non-empty raw; defaults do NOT count.

function rule<const D extends readonly Tessera[]>(
  deps: D,
  fn: (values: RuleValues<D>) => Issue[] | void,   // values: positional, aligned to deps;
  meta: { id: string; description: string },        //   an absent OPTIONAL dep is undefined
): Rule;                                             //   (a required dep can't be — bind fails first)
// Return Issue[] to fail — each Issue carries { path, message }, richer than a boolean, and
// redacts to counts+paths for secret deps; void/[] passes. Runs at bind and at every
// snapshot()/pin — never at unwrap(); promotes deps to eager.
```

Discriminants and `when()` condition tesserae must be `freshness: "snapshot"`
(`LiveDiscriminantError` at construction and at bind). Relations round-trip as recursive
tagged JSON (`relv1|` content-addressed); nesting is rejected in v1
(`VariantsNestingError`). A variants group is client-**ineligible** in v1
(`VariantsInClientTargetError`). Full semantics: [Conditional config](/docs/concepts/conditional).

## Async bindings and writes — `tessellum/async`

```ts
import { bind } from "tessellum/async";

type ReadSlot =
  | { raw: Json; version?: string; cacheUntil?: number }  // present; cacheUntil = CACHE ceiling:
  | { absent: true }                  //   effective staleness = min(maxStalenessMs, cacheUntil - now)
  | { error: unknown };               // per-key store failure (final — provider retries transients)

interface AsyncProvider {
  id: string;                 // stable identity — recorded in the lockfile (anonymous impls error on bake-covered targets)
  color: "async";             // declared, never guessed — untyped color is a compile error
  limits?: { maxReadBatch?: number; maxWriteBatch?: number };
  read(keys: readonly string[]): Promise<readonly ReadSlot[]>;
  // BATCH-ONLY, input-aligned: result[i] answers keys[i]; length mismatch = ProviderContractError.
  write?(entries: ReadonlyArray<{ key: string; raw: Json; ifVersion?: string }>):
    Promise<ReadonlyArray<{ version?: string; storedRaw?: Json }>>;
  capabilities?: {
    versionTokens?: "unique" | "coarse" | "none";  // timestamps are NOT versions
    strongRead?: boolean; echoWrite?: boolean; normalizes?: boolean;
    atomicWrites?: boolean;      // batch write = one transaction, all-or-nothing incl. CAS abort
    consistentReads?: boolean;   // one read() within maxReadBatch = one consistent cut
  };
}
// Core guarantees: unique keys, never-empty batches, chunking is CORE's job (provider-
// internal chunking voids consistentReads), duplicate write keys rejected pre-I/O, and
// multi-entry writes go ONLY to atomicWrites providers — otherwise core sequences singles
// in dependency order; an atomic write is ONE write() call and never spans chunks, so a
// setMany past maxWriteBatch is refused pre-I/O (AtomicBatchTooLargeError, ADR 0006).
// Per-key {error} slots route to a per-key unavailable path; a
// whole-call throw = connection-level down. consistentReads is demoted to a round-trip
// optimization: falsy/chunked coherent reads run a bounded coherence re-check loop
// (discriminant or coherent() group; CoherenceError past the retry budget) — see ADR 0004.
// A published verifyProvider(impl) conformance kit is the bar for declaring capabilities.

// SECRET PROVIDERS — same batch shape, class pinned (see the Secrets page):
interface SecretProvider {
  id: string; color: "async";
  class: "secret";                     // general providers: class?: "config" (default)
  limits?: { maxReadBatch?; maxWriteBatch?; maxConcurrentBatches? }; // reads fan out in
                                       //   parallel bounded by maxConcurrentBatches (~4);
                                       //   writes stay sequential, dependency-ordered
  read(keys: readonly string[]): Promise<readonly ReadSlot[]>;      // SHARED slot type
  write?(entries): Promise<readonly SecretWriteResult[]>;
  // SecretWriteResult = Omit<WriteResult, "storedRaw"> — DERIVED: an author cannot echo
  // a secret; capabilities omit echoWrite. Core also runtime-reshapes every slot/result
  // from secret-serving providers (whitelist copy) — the type is the lint, the reshape
  // is the enforcement.
  capabilities?: { versionTokens?; strongRead?; normalizes?; atomicWrites?; consistentReads? };
}
// PAIRING (bind + bake, lockfile {implId, class, holdsSecrets} normative): a secret-class
// provider serves ONLY secret-exposure tesserae; a provider-sourced SECRET tessera requires a
// secret-class provider OR a general provider declaring holdsSecrets: true — which
// inherits the FULL secret obligation set (error QUARANTINE — provider error objects are
// never propagated; result reshape; secret conformance profile; echo-verify disabled) and
// exposure-SEGREGATED batches. env/file-sourced secrets are unaffected. Secret writes
// verify by version equality > strongRead digest > honest "inconclusive" outcome
// (per-tessera requireVerifiedWrites refuses). literal() on a secret tessera = SecretLiteralError
// outside the test entry (secretLiteral()/fakeSecretProvider() live in the testing kit).
// Variants discriminants must be exposure "public" (DescriptorError at construction).

interface AsyncBinding {
  ready(): Promise<void>;                        // startup line — one batch read, validates all; boot outage → bounded retry then fail-fast (readyTimeoutMs)
  unwrap<T>(k: Tessera<T>): Promise<T>;             // colored read (scripts/jobs)
  safe<T>(k: Tessera<T>): Promise<{ ok: true; value: T } | { ok: false; error: ConfigReadError }>;
  snapshot(): Promise<Snapshot>;                 // per-request pin — re-reads expired keys; a coherent() group re-reads whole, as one cut
  set<T>(k: Tessera<T>, raw: unknown,               // input-typed: validate raw → write verbatim → cache PARSED → epoch++
         opts?: { ifVersion?: string; verify?: "off" | "report" | "revert" }): Promise<SetResult>;
  setMany(entries): Promise<SetResult[]>;        // post-state validated as a whole; atomic on atomicWrites providers (≤ maxWriteBatch, else AtomicBatchTooLargeError)
  refresh(): Promise<Snapshot>;                  // re-reads discriminants + full active branch as one coherent batch
  epoch(k: Tessera): number;
  health(): HealthReport;
}

const cfg = bind({ dbHost: provider(dbHost, pg), dbPassword: provider(dbPassword, pg), flags: provider(flags, pg) },
                 { maxStalenessMs: 5_000,
                   coherent: [[dbHost, dbPassword]] });   // async entries → AsyncBinding;
                                       // coherent: rotate-together keys pinned & re-read as ONE cut
await cfg.ready();        // THE startup line — one batch read; store down at boot → bounded retry (backoff), then SourceUnavailableError past readyTimeoutMs
await cfg.unwrap(k);      // colored read (scripts/jobs); request paths pin instead:
await cfg.snapshot();     // zero-I/O if fresh; re-reads expired keys, and any coherent() group whole
await cfg.set(k, raw, { ifVersion, verify });  // input-typed: validate raw → write raw
                                       // verbatim → install PARSED in cache (read-your-writes) → epoch++
// verify: "off" (default) | "report" | "revert" → SetResult
//   { version, outcome: "installed" | "verified" | "inconclusive" | "superseded", reason? }
// verified via write-echo (storedRaw, same round trip) or strong read-back; equality is
// schema-level (byte-equal OR structurally-equal parsed). Confirmed mangling (version-
// matched, content different) throws WriteMangledError { revertWritten, revertVerified?,
// nowLive }. "revert" is capability-gated (unique version tokens + echoWrite/strongRead,
// CAS implied, prev must exist and re-validate, secrets refuse unless opted in) —
// VerifyCapabilityError at call time otherwise. Never reverts on inconclusive.
await cfg.setMany(entries);            // post-state validated as a whole; atomic ≤ maxWriteBatch (else AtomicBatchTooLargeError; or { allowNonAtomic })
await cfg.refresh();      // re-reads discriminants + full active branch as one coherent batch
```

`set()` takes the schema's **input** type — Standard Schema has no inverse, so parsed
values can never round-trip. Expired reads block on a per-key single-flight re-read:
`maxStalenessMs` is the honest *staleness* bound (how old a served value may be) and
`readTimeoutMs` is the *wait* bound (default `maxStalenessMs`) — a blocking re-read that
breaches it throws `SourceUnavailableError` rather than wait unbounded or serve a value past
its staleness budget ([ADR 0005](https://github.com/rejifald/tessellum/blob/main/docs/adr/0005-async-read-deadline-and-boot-outage.md)).
`ready()` has no cache to fall back on, so a store outage at boot is bounded retry with
backoff, then fail-fast past `readyTimeoutMs`. **Cross-key coherence within a pin is opt-in:**
the `coherent` bind option names tesserae that must not tear across a generation boundary — a
rotated `dbHost` + `dbPassword` pair — and `snapshot()` re-reads the whole group as one cut
whenever any member is expired (one `read()` when it fits a batch and the provider declares
`consistentReads`, the bounded re-check loop otherwise; a slow group re-read is itself bounded
by `readTimeoutMs`, a churning one throws `CoherenceError`). A group must resolve to a single
provider or bind fails with `CoherenceGroupSourceError`; ungrouped keys stay
expired-keys-only, with no cross-key promise. A variant's discriminant + active branch is an
implicit group (`refresh()` reads it as one coherent batch). See
[ADR 0004](https://github.com/rejifald/tessellum/blob/main/docs/adr/0004-async-snapshot-coherence.md).
Discriminants pin at `ready()`; `set()` on one is a
*staged* write observed only via `refresh()`. Scope line: pull-only, context-free,
uniform, operator-plane writes. Full walkthrough:
[Database-backed config](/docs/guides/database-config).

## Identity

```ts
function identityOf(k: Tessera): string;   // "tesserav1:xxh128:…"
```

Identity is content-addressed over a **closed field list**:
`key`, `exposure`, `sources` (order-significant), `default` (presence + value),
`freshness` (live-vs-snapshot is behaviorally load-bearing — relations reference
discriminants by identity), and
`schemaId` when available (vendor fingerprint, else the explicit `version` field).
`doc`, examples, and all unknown fields are cosmetic. The preimage is `tesserav1|` +
RFC 8785 canonical JSON, hashed with xxh128 — the hash is an index, not a security
boundary; structural equality of the identity subset is authoritative.

Consequences:

- Duplicated installs of the *same* descriptor (dual-package hazard, diamond deps) resolve
  interchangeably — and hash-fallback resolutions always **re-validate with the caller's
  schema**, so validation can never silently downgrade across package copies.
- The same key under a *different* descriptor is `DescriptorMismatchError`, carrying both
  canonical descriptors, a field-level diff, and a paste-able `unify` snippet.

## Testing

```ts
import { bind, literal } from "tessellum";   // core bind — the purest test spelling

const cfg = bind({ db: literal(dbUrl, "postgres://localhost:5432/test") });
// input-typed, still validated (a bad test value fails loudly), zero process.env mutation
// Never commit real secret values — placeholders only; bake fails closed on
// literal-wired secrets in client targets.
```

The **testing kit** is a dedicated `tessellum/testing` entry point of pre-certified doubles:
`secretLiteral()` (the secret-safe `literal()` — test-scoped, so `SecretLiteralError` still
guards production), `fakeSecretProvider()` (an in-memory `SecretProvider` that already
passes the conformance kit), `manualClock` (drive `maxStalenessMs` and coast budgets
deterministically), and `verifyProvider(impl)` (the conformance kit itself — the bar for
declaring a provider's `atomicWrites` / `consistentReads`).

## Bake — the CLI

```ts
// config.manifest.ts — side-effect-free by contract; bake executes it in a
// constrained context (tessera construction is inert, so I/O here is detectable)
export default {
  targets: {
    client: [...appPublicTesserae, ...widgetTesserae], // app attestation: deny-by-default
    server: [...allTesserae],
  },
};
```

```bash
tessellum bake           # generate per-target modules + .env.example + CONFIG.md + manifest.json + leak scan
tessellum bake --check   # CI drift gate
tessellum bake --watch   # dev / HMR (dev output is unscanned — banner says so)
```

Per target:

- **client** → `config.baked.ts`: public literal values plus an inlined ~30-line snapshot
  shim that imports nothing at runtime (zero runtime dependency), carries the `Scope`
  brand and per-tessera identity hashes, and throws `StaleBakeError` on bake/runtime skew.
  A secret tessera in `targets.client` fails the bake.
- **server** (optional wiring) → dynamic computed env access (`process.env[keys[i]]`),
  never static `process.env.KEY` (bundler `define` folds static member expressions into
  literal secrets), and never values.

Bake **never resolves secret values** — public values and secret key *names* only.
Secrets validate at runtime, where the value actually lives. See
[Bake](/docs/bake/overview) and the [leak scan](/docs/bake/leak-scan).

## Error taxonomy

Core: `MissingConfigError` · `UnboundTesseraError` · `DescriptorMismatchError` ·
`NotInScopeError` · `ExposureError` ·
`AsyncValidatorInSyncBindingError` · `NoValidValueError` · `StaleBakeError`.
Live: `InvalidLiveValueError` · `SourceUnavailableError` · `CoastBudgetExceededError`.
Variants: `InactiveVariantError` (subclass of `UnboundTesseraError`) ·
`MissingDiscriminantError` · `LiveDiscriminantError` · `AmbiguousTesseraMembershipError` ·
`VariantsNestingError` · `RuleViolationError` · `VariantsInClientTargetError`.
Writes: `WriteConflictError` · `NotWritableError` · `WriteFailedError` ·
`WriteMangledError` · `VerifyCapabilityError` · `ProviderContractError` ·
`AtomicBatchTooLargeError`.
Shapes & wiring: `LivenessViolationError` · `ReservedKeyError` · `UnknownWiringError` ·
`WiringCollisionError` · `WiringExposureError` · `WiringSkewError` ·
`DuplicateWiringError` · `NoWiringError`.
Secret plane: `SecretLiteralError` · `ExposureMigrationError` · `DescriptorError`.
Every error names the tessera key, the layer, and a next step. For secret tesserae, BYO
validator text is untrusted (validators echo received values): reports carry issue counts
and paths only. Full details: [Error taxonomy](/docs/reference/errors).
